## 5.0

* Features
  * Implemented encryption
    * The program uses cryptography fernet to handle encrypting and decrypting data
    * passwords are encrypted when being added to the database
  * Implemented a login system
    * The user must login into the database to allow recipe editing
* Created the cryptography test to learn how to use fernet
* Recipe App
  * added byte key & logged in values to handle encryption and signing in
  * new functions
    * generate byte key - Creates and returns a randomly generated byte key
    * reset byte key - Sets the byte key to empty
    * get byte key - returns the byte key but returns a value error if the byte key is b''
    * save byte key(path) - saves the byte key to a text file at a given path
    * load byte key(path) - loads the byte key from a text from a given path
    * get fernet - returns a fernet object used for encryption and decryption
  * the create database and load database now use the InitDatabaseScreen
* LoadDatabaseScreen is now deprecated due to the InitDatabaseScreen replacing the screen
* CreateAdmin screen & Login screen now have error labels and have functionality
* CreateDatabaseScreen
  * Now called InitDatabaseScreen
  * has a load and create mode
  * added a key path variable
  * now has a new button for selecting the key path
  * when initializing the file paths, it can either create a new key file or load a key file
  * when creating the user is sent to the create admin screen. If the loaded database has no user, the program will send the user to the create admin screen
* You can now login using the login screen
* The main menu will update the button to allow sign out or login based on if the user is logged in
* Login changes
  * The create recipe button is disabled in the index screen if the user isn't logged in
  * Many user inputs in the view recipe index are disabled if the user isn't logged in
